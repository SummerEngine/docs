## AI Integration via MCP

Summer Engine's AI is accessed through the **MCP (Model Context Protocol)** integration. The MCP server exposes 44 tools that let any MCP-compatible AI tool control the engine: add nodes, set properties, import assets, run the game, and debug.

This is the same interface used by:
- **Summer Agent** (built-in chat in the editor)
- **Cursor**, **Claude Code**, **Windsurf**, **Antigravity**, **Codex**, **VS Code**, and **Zed** via MCP

## How It Works

1. Summer Engine runs a local API server on `localhost:6550` when a project is open
2. The MCP server (`npx summer-engine mcp`) connects to this local API
3. Your IDE's AI uses the 44 MCP tools to manipulate the engine

No cloud API keys needed for MCP. The engine handles authentication locally via `~/.summer/api-token`.

## Example: Add a Node via MCP

When you ask your AI IDE "Add a Camera3D to the scene," the AI calls:

```
summer_add_node(parent="./", type="Camera3D", name="MainCamera")
```

The node appears in Summer Engine immediately.
